## Product Overview

FairSplit is a zero-friction shared expense tracker for friend groups, couples, and roommates. Members join via a shareable link with just a display name — no account, no email, no password. They log expenses as they go, see a running balance for every member, and resolve all debts in one tap via a minimum-transfer settle-up algorithm that computes the optimal payment plan. Completely free, zero external dependencies, runs entirely with `docker compose up`.

## Session Model (critical — read carefully)

Two-layer session for anonymous members:
- **Layer 1 — Cookie (authoritative):** Server sets `fairsplit_member_token` as `httpOnly`, `Secure`, `SameSite=Lax`. Safari ITP does NOT clear httpOnly cookies. 30-day rolling expiry.
- **Layer 2 — localStorage (read cache):** JWT also returned in response body. Frontend stores as `fairsplit_token_{group_id}`. If localStorage is cleared by ITP, call `GET /api/v1/groups/{id}/members/me` (uses cookie) to recover — **never redirect to join screen just because localStorage is empty**.

JWT payload: `{sub: member_id, group_id, is_admin, iat, exp}`

## Settle-Up Algorithm (hero feature — correctness is zero-tolerance)

Greedy max-heap approach, O(n log n). Computed at read time — not stored.

**Required unit test cases (all must pass before merge):**
1. Two-person single debt → exactly 1 transfer
2. Three-person circular debt → exactly 2 transfers (not 3)
3. Four-person complex → minimum transfers verified manually
4. All-settled state → empty transfer list
5. 1-cent remainder → assigned to first creditor alphabetically
6. 20-member stress → completes in < 50ms

Service: `src/backend/services/settle_service.py` → `compute_settle_up(net_balances, member_names, currency)`

## Key User Flows

1. **Create & share:** User creates a group → gets a shareable link → sends to friends
2. **Join (no account):** Friend opens link → enters display name → lands on dashboard in < 60s
3. **Log expense:** Any member logs who paid, amount, split type → balances update instantly
4. **Settle up:** Group taps Settle Up → sees minimum-transfer plan → copies to clipboard → marks transfers as settled

## Critical Implementation Rules

- **All money is integer cents.** `amount_cents: int`. Never float. Never Decimal in API responses. `$42.00 = 4200`.
- **Idempotency on mutations.** `POST /expenses` and `POST /groups` require `X-Idempotency-Key` (client UUID). Check `idempotency_keys` table before processing.
- **Amount inputs use `inputMode="decimal"` + `type="text"`** — not `type="number"`. Prevents iOS step-validation issues.
- **Fetch calls use `credentials: "include"`** — required to send httpOnly cookie cross-origin.
- **WCAG primary button colour: `#047857` (emerald-700)**, not `#10b981` (emerald-500). emerald-500 fails AA contrast for white text.
- **Draft persistence:** `add-expense` form state saved to `localStorage` on `beforeunload`. Restored on mount. Key: `fairsplit_draft_{group_id}`.
- **Balance polling:** SWR `refreshInterval: 10000`. No loading indicator for background refreshes — skeleton only on initial load.
